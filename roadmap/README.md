# Learning Roadmap

A sequenced path for scientists who want to build solid software engineering (SE) foundations.
Resources are ordered from zero to advanced — start at the top and work your way down.

---

### 1. Head First Software Development
📖 Book | **Andrew Stellman & Jennifer Greene** | Beginner

A visual, beginner-friendly introduction to software development practices using real-world
scenarios — ideal if you have never thought about code as a designed artifact.

> Start here, especially if the words "sprint", "refactor", or "version control" feel unfamiliar.

---

### 2. Introduction to Git and GitHub
🎓 Course | **Google Career Certificates** | Beginner | [Coursera](https://www.coursera.org/learn/introduction-git-github)

Hands-on introduction to version control with Git and GitHub — branches, merges, remote
repositories, and collaborative workflows. No prior experience needed.

> Take this **alongside #1** to get practical Git fluency before adopting it as your lab notebook.

---

### 3. The Pragmatic Programmer
📖 Book | **David Thomas & Andrew Hunt** | Beginner–Intermediate

Timeless habits and attitudes of effective software practitioners — how to think about
your code, your tools, and your career as a scientist who writes software.

> Read this **after finishing your first real project** — the advice will make immediate
> sense once you have felt the pain it describes.

---

### 4. Modern Software Engineering
📖 Book | **David Farley** | Intermediate

The theoretical foundation for this workshop: why software engineering is a discipline,
and how to optimize for learning and managing complexity.

> Read this **after the workshop** to deepen the theory behind every concept covered.

---

### 5. Clean Code
📖 Book | **Robert C. Martin** | Intermediate

How to write code that humans can read, understand, and change — naming, functions,
and structure at the line and module level.

> Read this **once you have working code** and want to make it readable and maintainable
> for your future self and collaborators.

---

### 6. Unit Testing: Principles, Practices and Patterns
📖 Book | **Vladimir Khorikov** | Intermediate

How to write tests that are genuinely valuable and maintainable — not just tests that pass,
but tests that give you real confidence in your code.

> Read this **when you start writing tests** and want to understand what separates a useful
> test from a fragile one.

---

### 7. A Philosophy of Software Design
📖 Book | **John Ousterhout** | Advanced

How to manage complexity through deep modules, simple interfaces, and deliberate
information hiding — a rigorous way to think about system design.

> Read this **after building several projects** — it rewards readers who have already felt
> the pain of systems that became too complex to change.
